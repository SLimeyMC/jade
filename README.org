* Jade Language

Jade is a minimal Lisp dialect for extensible configuration systems where code and data are indistinguishable. Most features are evaluator-provided rather than built into the language itself.

** Core Syntax

Reader macros:

: 'x     -> (quote x)
: `x     -> (quasiquote x)
: ,x     -> (unquote x)
: ,@x    -> (unquote-splice x)
: @x     -> (tag x)
: #x     -> (hash x)
: $x     -> (ref x)
: $x.y   -> (ref x y)
: :x     -> (on x)

The reader only parses strings, pairs, and nil. Numbers, booleans, and other literals are readily-defined symbols. Symbols with spaces use ~|hello world|~.

** Architecture

Jade has three stages:

1. Read :: Parse expressions and expand reader macros into AST.
2. Analysis :: Optional validation such as type checking or arity checking.
3. Evaluation :: Reduce expressions into simpler forms.

Example:

: (= 1 (+ 2 -1))
: (= 1 1)
: t

: (let x :int 1)
: (var y :int 1)
: (set! y 1)

: (lambda (param) body)

: (list 1 2 3)
: (map (a 1) (b 2))

: (when
:   (cond1 expr1)
:   (cond2 expr2))

Type examples:

: :[_]:int
: :[str]:int

** Evaluation Model

Jade emphasizes declarative evaluation and stack-local allocation. Functions are intended to be total and pass-by-value.

Example:

#+BEGIN_SRC lisp
(let +2
  (fun (:int arg1)
    (+ arg1 2)))

(+2 1) ;; => 3
#+END_SRC

** Quoting

Quote escapes evaluation:

: x    ;; evaluated
: 'x   ;; literal symbol

Quasiquote supports partial evaluation:

#+BEGIN_SRC lisp
(let x '(2 3))

`(1 ,x 4)   ;; => (1 (2 3) 4)
`(1 ,@x 4) ;; => (1 2 3 4)
#+END_SRC

** Runtime References

Runtime-provided bindings use ref:

: (ref window)
: (set! $window.size (width height))

** Design Notes

Special prefixes such as ~@~, ~#~, ~$~, and ~:~ are parsed into explicit unary forms rather than embedded into symbols. This keeps evaluator behavior predictable and simplifies parsing.
