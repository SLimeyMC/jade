* Outline
Jade are dialect of Lisp. To make an extensible configuration file where data and code is indistinguishable. Currently it is in production and most if feature hasn't been implemented. This README serve as what could be of the language once everything is implemented.
The only macro that exist in Jade are

: 'value -> (quote value)
: `value -> (quasiquoute value)
: ,value -> (unqoute value)
: ,@value -> (unquote-splice value)
: @value -> (tag value)
: #value -> (hash value)
: $value -> (ref value)
: $value.subvalue -> (ref value subvalue)
: :value) -> (on value)
/currently reader don't parse ~#~, ~,@~, ~@~, ~$~, :/

The only literal reader can parse is string, pair, and nil. Integer, boolean exist but they are part of the evaluator, say you write true, the evaluator will parse it as string of "true" but evaluator can have ability of instead parsing it as boolean true, think of it as javascript string interpolation. List, map and so on are using the same expression so ~(List 1 2)~ or ~(Map (a 2) (b 1))~. Sometimes symbol needed space, in this case wrap the symbol with ~||~, for example ~|hello world|~, use ~\|~ to escape. There are /no/ interpolation or templating inside the symbol unless the library provided it.

Jade are made to be as small as possible, and that the evaluator can be injected with as many feature as possible. The smallest possible feature is just printing out a single symbol, parentheses with multiple symbol doesn't even work without the quoting, which need to be added to the evaluator.

Jade are divided into 3 step. Read, parsing the expression while expanding macro, they are stateless and won't do any checking outside of parsing (in this case turning code into AST, not compiling). Analysis, checking the correctness of the code, the analysis will check for number of argument or type of each argument (if provided, type are optional and library can opt to not do it) [[read more][]]. Then there are evaluator, think of it as the runtime that evaluate the expression into it's smallest possible expression. For example

: (= 1 (+ 2 -1)) ;; the base
: (= 1 1) ;; the + evaluate 2 + -1 returning 1
: t ;; the = evaluate 1 is equal to 1 so true which is symbolized as t

* Keyword
While Jade doesn't need keyword, there are built in provided by the library. For example:

: (let x :int 1) ;; declare a variable of x with value of 1
: (var y :int 1) ;; declare a mutable variable of x with value of 1
: (set! y 1) ;; set y with value of 1
: (ref window) ;; referenced a global variable of z (provided by runtime)
: (cond (cond1 expr1) (cond2 expr2) ;; a chained if where it run when the condition are met
: (lambda (param) body) ;; a lambda declaration
: (list 1 2 3) ;; construct a list backing which evaluated into expression of (ref <name>)
: (map (a 1) (b 2)) ;; construct a list backing which evaluated into expression of (ref <name>)
Also to type for list it would be :[_]:int and map :[str]:int.

: ;; operator like + - * / where it run from left to the right
: ;; operator like = > < <= >= where it check equality for each symbol with && for each succesor to it's predecessor

There are other keyword that is sensitive to context, they are evaluated as is other than the exception:
: (tag |this is doc|) ;; tag can be think of as tagging something.
: (empty) ;; empty can be think of empty list (because () is equal to nil), although this keyword are sensitive to context.
: (else cond2) ;; else here for cond block
The other ~hash~, and on are also keyword that is sensitive to context, and there are no
sensible default/consensus on which one should apply, it should be up to the library to handle it.

* Evaluation
In Jade, the only allocation that happen are stack local. There are no borrowing or anything, because concept of function
must be total. This is to ensure declarativity of the configuration. For example:
#+BEGIN_SRC code_1
jade>(fun +2 :int arg1 :int @|Addition of two integer|
	>	(+ arg1 2))
jade> (+2 1)
  ~~> 3
jade>;; Here the +2 are wired to previously declared symbol of +2 which is function accepting arg1, that is 1 and return 3
jade>(let x 1)
jade>(let y 1)
jade>(= x y)
  ~~> true
jade>(fun check a b
	>	(let a (+2 a))
	>	(+ a b x)) ;; here no borrowing happen, must set! and that a create new stack local variable, ignoring parameter a and x reference global x
jade>;; example
	>(fun set-window width height
	>	(set! $window.size (width height))
#+END_SRC

** Quoting
In jade we have quote to escape evaluation say

: (let x 2)
: x ;; become 2
: 'x ;; however it is just x

But then something like ~`~ and ~,~ which is a partial escape where ~,~ signal to evaluate the expression. The ~,@~
signal to inject the evaluation meaning
: (let x '(2 3))
: `(1 ,x 4) ;;-> (1 (2 3) 4)
: `(1 ,@x 4) ;;-> (1 2 3 4)

* FAQ

Q: Why not make the special symbol (~@~, ~#~, ~$~, ~:~) part of symbol instead of being parsed as pair to it's value?
A: This is to make the evaluator more decisive on choosing what to do. Instead of parsing for the ~@~ the reader attach
it as unary to the value. The syntax becoming more concise and has clear standard for evaluator to follow.

Q: What is ~(ref x)~?
A: ~ref~ is a flag to reference to x. Jade provide a binder, where evaluator can provide a structured data. And then
Jade match and inject with the proper symbol provided.
